Documento de Sustentación

Introducción

Este documento tiene como propósito exponer la justificación y sustentación de la elección de la herramienta de software que se implementará en el área de gestión de proyectos de nuestra empresa. La elección de la herramienta está basada en un análisis exhaustivo de diversas opciones y se seleccionó la que mejor se adapta a nuestras necesidades actuales y futuras.

Herramientas utilizadas

Este proyecto se basa en una metodología que aprovecha el motor de plantillas Jinja2, permitiendo la creación dinámica y eficiente de contenido HTML en el frontend a partir de datos proporcionados desde el backend. Las herramientas empleadas en el desarrollo incluyen tecnologías clave para garantizar un flujo de trabajo optimizado y una experiencia de usuario interactiva y adaptable; las herramientas empleadas en el proyecto son:

  1. Python: Como lenguaje base para el backend, brindando capacidad de procesamiento y manejo de lógica de negocio.
  2. Flask: Framework web minimalista para gestionar rutas, solicitudes HTTP y la integración entre el backend y las vistas en HTML.
  3. HTML, CSS y JavaScript: Tecnologías de frontend para estructurar, diseñar e interactuar con los elementos visuales de la aplicación.
  4. Bootstrap: Esta biblioteca de diseño front-end facilitó la creación de una interfaz responsiva, estéticamente atractiva y compatible con múltiples dispositivos, optimizando el diseño sin comprometer el rendimiento.
  5. VS Code: VS Code fue el entorno de desarrollo principal, ofreciendo una experiencia optimizada con extensiones específicas para Python, Flask y herramientas de depuración, lo cual mejoró la eficiencia en el desarrollo y mantuvo la calidad del código.

Conclusión

Tras realizar un análisis detallado de las opciones disponibles, hemos determinado que la herramienta "ProjectMaster" es la mejor opción para nuestras necesidades. Esta herramienta ofrece una interfaz intuitiva, tiene un conjunto de funcionalidades completo y su costo es competitivo. Además, el soporte al cliente de "ProjectMaster" ha sido calificado como excelente por otros usuarios.

"La selección de una herramienta adecuada para la gestión de proyectos no solo optimiza los procesos, sino que también facilita la colaboración entre los miembros del equipo." - Equipo de TI
